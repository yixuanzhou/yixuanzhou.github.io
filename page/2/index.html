20. Valid Parentheses

Posted on 2019-01-10 | In LeetCode

Problem Description

Given a string containing just the characters ‘(‘, ‘)’, ‘{‘, ‘}’, ‘[‘ and ‘]’, determine if the input string is valid.

An input string is valid if:

Open brackets must be closed by the same type of brackets.
Open brackets must be closed in the correct order.
Note that an empty string is also considered valid.

Example 1:

Input: "()"
Output: true

Example 2:

Input: "()[]{}"
Output: true

Example 3:

Input: "(]"
Output: false

Example 4:

Input: "([)]"
Output: false

Example 5:

Input: "{[]}"
Output: true

Note

Typical stack problem.

Code

public boolean isValid(String s) {
    Stack<Character> stack = new Stack<>();

    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '[') stack.push(']');
        else if (c == '{') stack.push('}');
        else if (stack.isEmpty() || stack.pop() != c) return false;
    }

    return stack.isEmpty();        
}

103. Binary Tree Zigzag Level Order Traversal

Posted on 2019-01-10 | In LeetCode

Problem Description:

Given a binary tree, return the zigzag level order traversal of its nodes’ values. (ie, from left to right, then right to left for the next level and alternate between).

For example:
Given binary tree [3,9,20,null,null,15,7],

    3
   / \
  9  20
    /  \
   15   7

return its zigzag level order traversal as:
[
  [3],
  [20,9],
  [15,7]
]

Note

Use a flag to control the direction.

Code

public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
    List<List<Integer>> all_lvls = new ArrayList<>();
    if (root == null) return all_lvls;
    LinkedList<TreeNode> curr_q = new LinkedList<>();
    curr_q.add(root);
    boolean dir = true;

    while (curr_q.size() != 0) {
        List<Integer> curr_lvl = new ArrayList<>();
        LinkedList<TreeNode> next_q = new LinkedList<>();
        while (curr_q.size() != 0) {
            TreeNode curr = curr_q.remove();
            if (dir) curr_lvl.add(curr.val);
            else curr_lvl.add(0, curr.val);
            if (curr.left != null) next_q.add(curr.left);
            if (curr.right != null) next_q.add(curr.right);                
        }
        all_lvls.add(curr_lvl);
        curr_q = next_q;
        dir = !dir;
    }

    return all_lvls;        
}

151. Reverse Words in a String

Posted on 2019-01-09 | Edited on 2019-01-10 | In LeetCode

Problem Description:

Given an input string, reverse the string word by word.

Example:  

Input: "the sky is blue",
Output: "blue is sky the".

Note:
A word is defined as a sequence of non-space characters.
Input string may contain leading or trailing spaces. However, your reversed string should not contain leading or trailing spaces.
You need to reduce multiple spaces between two words to a single space in the reversed string.

Note

Use a StringBuilder, iterate the string from back to front and select word to be appended to StringBuilder.

Time complexity: O(n).

Code

public String reverseWords(String s) {
  StringBuilder res = new StringBuilder();

  for (int i = s.length()-1; i > -1; i--) {
      if (s.charAt(i) == ' ') continue;
      int start = i;
      while (start > -1 && s.charAt(start) != ' ') start--;
      res.append(s.substring(start+1, i+1)).append(" ");
      i = start;
  }

  return res.toString().trim();
}

653. Two Sum IV - Input is a BST

Posted on 2019-01-09 | In LeetCode

Problem Description:

Given a Binary Search Tree and a target number, return true if there exist two elements in the BST such that their sum is equal to the given target.

Example 1:

Input:
    5
   / \
  3   6
 / \   \
2   4   7

Target = 9
Output: True

Example 2:

Input:
    5
   / \
  3   6
 / \   \
2   4   7

Target = 28
Output: False

Note

Typical Two Sum. Use a stack to iterate through all nodes and add node’s value into a list. Just check if list contains (target-current).

Time complexity: O(n).

Code

public boolean findTarget(TreeNode root, int k) {
    if (root == null) return false;
    List<Integer> list = new ArrayList<>();
    Stack<TreeNode> stack = new Stack<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode curr = stack.pop();
        if (list.contains(k-curr.val)) return true;
        list.add(curr.val);
        if (curr.left != null) stack.push(curr.left);
        if (curr.right != null) stack.push(curr.right);
    }
    return false;
}

206. Reverse Linked List

Posted on 2019-01-07 | In LeetCode

Problem Description:

Reverse a singly linked list.

Example:

Input: 1->2->3->4->5->NULL
Output: 5->4->3->2->1->NULL

Note

Iterative method just move the element that curr.next points to (the initial head of the list) to become the new head, illustration:

head curr back
1 2 3 4 5 1 3 4 5 2 3 4 5 -> 2 1 3 4 5
2 1 3 4 5 1 4 5 3 4 5 -> 3 2 1 4 5
3 2 1 4 5 1 5 4 5 -> 4 3 2 1 5
4 3 2 1 5 1 5 -> 5 4 3 2 1
5 4 3 2 1

Recursive method is pretty straight-forward, each time we take the head out of list and reverse the remaining list, then set head to be the remaining.next and set head.next to null.

Time complexity: O(n).

Code

// Recursive version
public ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode tmp = head.next;
    ListNode res = reverseList(tmp);
    tmp.next = head;
    head.next = null;
    return res;
}

// Iterative version
public ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode curr = head;
    ListNode back = null;
    while (curr.next != null) {
        back = curr.next;
        curr.next = curr.next.next;
        back.next = head;
        head = back;
    }
    return head;
}

340. Longest Substring with At Most K Distinct Characters

Posted on 2019-01-06 | In LeetCode

Problem Description:

Given a string, find the length of the longest substring T that contains at most k distinct characters.

Example 1:

Input: s = "eceba", k = 2
Output: 3
Explanation: T is "ece" which its length is 3.

Example 2:

Input: s = "aa", k = 1
Output: 2
Explanation: T is "aa" which its length is 2.

Note

Almost the same as LC159.
Use a HashMap to keep track of frequency of each character, if map size exceeds k, deduct the frequency of character pointed by left pointer by 1 and move forward the left pointer. Remove the character from map if the frequency reaches 0;

Time complexity: O(n).

Code

public int lengthOfLongestSubstringTwoDistinct(String s) {
    int left = 0, right = 0;
    int maxLength = 0;
    Map<Character, Integer> window = new HashMap<>();

    while (right < s.length()) {
        char curr = s.charAt(right);
        window.put(curr, window.getOrDefault(curr, 0)+1);
        while (window.size() > k) {
            char prev = s.charAt(left++);
            window.replace(prev, window.get(prev)-1);
            window.remove(prev, 0);
        }
        maxLength = Math.max(maxLength, right-left+1);
        right++;
    }

    return maxLength;    
}

Example 2:

Use a HashMap to keep track of frequency of each character, if map size exceeds 2, deduct the frequency of character pointed by left pointer by 1 and move forward the left pointer. Remove the character from map if the frequency reaches 0;

Time complexity: O(n).

Code

public int lengthOfLongestSubstringTwoDistinct(String s) {

        char curr = s.charAt(right);

            window.remove(prev, 0);

239. Sliding Window Maximum

Posted on 2019-01-06 | In LeetCode

Problem Description:

Given an array nums, there is a sliding window of size k which is moving from the very left of the array to the very right. You can only see the k numbers in the window. Each time the sliding window moves right by one position. Return the max sliding window.

Example:

Input: nums = [1,3,-1,-3,5,3,6,7], and k = 3
Output: [3,3,5,5,6,7]
Explanation:

Window position                Max
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7

Note

For this problem, just use a doubly-linked list and two pointers. When iterating through nums array, add right pointer index into the list. The doubly-linked list keeps largest number within the k distance from right pointer. The rule for sliding window is as follows: if right pointer index is bigger than the index of header of the list, this means the sliding window has exceeded its size k, so to remove the head of list. If the current number is larger than previous the number added to the list, just poll out that number in the list.

Time complexity: O(n), Space complexity: O(n).

Code

public int[] maxSlidingWindow(int[] nums, int k) {
    if (nums == null || k <= 0) return new int[0];
    int left = 0, right = 0;
    int[] res = new int[nums.length-k+1];
    Deque<Integer> win = new ArrayDeque<>();

    while (right < nums.length) {
        while (win.size() != 0 && win.peek() < right-k+1) win.poll();
        while (win.size() != 0 && nums[win.peekLast()] < nums[right]) win.pollLast();
        win.add(right++);
        if (right-left == k) res[left++] = nums[win.peek()];
    }

    return res;        
}

929. Unique Email Addresses

Posted on 2019-01-05 | In LeetCode

Problem Description:

Every email consists of a local name and a domain name, separated by the @ sign.

For example, in alice@leetcode.com, alice is the local name, and leetcode.com is the domain name.

Besides lowercase letters, these emails may contain ‘.’s or ‘+’s.

If you add periods (‘.’) between some characters in the local name part of an email address, mail sent there will be forwarded to the same address without dots in the local name. For example, “alice.z@leetcode.com” and “alicez@leetcode.com” forward to the same email address. (Note that this rule does not apply for domain names.)

If you add a plus (‘+’) in the local name, everything after the first plus sign will be ignored. This allows certain emails to be filtered, for example m.y+name@email.com will be forwarded to my@email.com. (Again, this rule does not apply for domain names.)

It is possible to use both of these rules at the same time.

Given a list of emails, we send one email to each address in the list. How many different addresses actually receive mails?

Example 1:

Input: ["test.email+alex@leetcode.com","test.e.mail+bob.cathy@leetcode.com","testemail+david@lee.tcode.com"]
Output: 2
Explanation: "testemail@leetcode.com" and "testemail@lee.tcode.com" actually receive mails

Note

The solution for this problem is pretty straightforward.

Code

public int numUniqueEmails(String[] emails) {
    Set<String> uniqueEmails = new HashSet<>();
    for (String email : emails) {
        String[] names = email.split("@", 2);
        String localName = names[0];
        String domainName = "@"+names[1];
        localName = localName.replaceAll(".", "");
        int plusIndex = localName.indexOf('+');
        if (plusIndex == -1) { // no plus sign
            uniqueEmails.add(localName + domainName);
        } else {
            int nextPlusIndex = localName.indexOf('+', plusIndex+1);
            if (nextPlusIndex == -1) { // only 1 plus sign
                uniqueEmails.add(localName.substring(0, plusIndex)+domainName);
            } else {
                uniqueEmails.add(localName.substring(0, plusIndex)+localName.substring(plusIndex+1, nextPlusIndex)+domainName);
            }
        }            
    }
    return uniqueEmails.size();       
}

621. Task Scheduler

Posted on 2019-01-05 | In LeetCode

Problem Description:

Given a char array representing tasks CPU need to do. It contains capital letters A to Z where different letters represent different tasks. Tasks could be done without original order. Each task could be done in one interval. For each interval, CPU could finish one task or just be idle.

However, there is a non-negative cooling interval n that means between two same tasks, there must be at least n intervals that CPU are doing different tasks or just be idle.

You need to return the least number of intervals the CPU will take to finish all the given tasks.

Example:

Input: tasks = ["A","A","A","B","B","B"], n = 2
Output: 8
Explanation: A -> B -> idle -> A -> B -> idle -> A -> B.

Note

The key idea here is to use a PriorityQueue to keep track of the remaining executing times of each task. We give priority to the task with highest remaining executing times. Besides, I use a HashMap to keep track of the last executing time of each task in order not to violate the cooling interval policy.

Code

public int leastInterval(char[] tasks, int n) {
    Map<Character, Integer> map = new HashMap<>();
    for (char task : tasks) map.put(task, map.getOrDefault(task, 0)+1);

    PriorityQueue<Map.Entry<Character, Integer>> pq = new PriorityQueue<>(
        (a,b) -> a.getValue() != b.getValue() ? b.getValue() - a.getValue() : a.getKey() - b.getKey());
    pq.addAll(map.entrySet());

    Map<Character, Integer> clk = new HashMap<>();
    for (char task : map.keySet()) clk.put(task, -n-1);
    int timeStamp = -1;
    int completed = 0;
    int cnt = 0;
    List<Map.Entry<Character, Integer>> cache = new ArrayList<>();

    while (completed < tasks.length) {
        timeStamp++;
        int k = n + 1;

        while (k > 0 && !pq.isEmpty()) {
            Map.Entry<Character, Integer> top = pq.poll();
            char task = top.getKey();
            if (timeStamp - clk.get(task) > n) {
                clk.replace(task, timeStamp);
                top.setValue(top.getValue()-1);
                if (top.getValue() != 0) pq.add(top);
                completed++; k--; cnt++; break;
            }
            cache.add(top);
        }

        if (pq.isEmpty() && k == n+1) cnt++; // idle
        pq.addAll(cache); cache.clear();
    }

    return cnt;
}
